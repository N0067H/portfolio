# Portfolio Project Guide (CLAUDE.md)

> 이 파일을 프로젝트 루트의 `CLAUDE.md`로 저장하면, 이후 모든 작업에서 Claude가 자동으로 이 규칙을 따릅니다.

## 프로젝트 개요
백엔드 개발자 류승엽의 단일 HTML 포트폴리오. PDF 인쇄 호환성이 최우선. 반응형 (모바일/데스크톱). 산출물은 `portfolio.html` 단일 파일.

## ⚡ 토큰 효율 규칙 (반드시 준수)
이 파일은 계속 부분 수정됩니다. 전체 재출력을 피하고 부분 수정(str_replace 등)이 쉽도록 코드를 작성하세요.

1. **데이터-마크업 분리**: 모든 콘텐츠는 `<script id="data">` 안 JSON 배열로 분리. 마크업은 JS로 렌더링. → 내용 수정 시 JSON만 건드림.
2. **CSS 변수 집중화**: 색상/폰트크기/간격은 `:root`의 CSS 변수로만 정의. 하드코딩 금지. → 디자인 수정 시 변수 한 줄만 변경.
3. **섹션 마커 주석**: 각 섹션 위에 `<!-- ===== SECTION: about ===== -->` 같은 고유 검색 가능 주석. → 원하는 섹션 즉시 grep 가능.
4. **반복 구조 템플릿화**: 프로젝트 카드, 트러블슈팅 블록 등 반복 요소는 단일 템플릿 함수로. 복붙 금지. → 한 곳만 고치면 전체 반영.
5. **한 줄 한 속성**: 긴 인라인 스타일·체이닝 금지. 수정 단위가 작게.
6. **수정 시 전체 출력 금지**: 변경 요청 시 바뀐 블록/함수/변수만 출력. "여기를 이렇게 바꾸세요" 형식.
7. **클래스명 안정성**: 한번 정한 class/id는 함부로 바꾸지 않음. (부분 수정 시 앵커 유지)

## 디자인 스펙
```
--bg: #F5F1E8;        /* 크림 (배경) */
--accent: #2D4A3A;    /* 어두운 녹색 (포인트) */
--accent2: #D4A574;   /* 갈색 (강조) */
--text: #2C2C2C;
--divider: #D4C9BC;
/* 다크모드 */
--bg-dark: #1A1A1A;
--text-dark: #E8E8E8;
--accent-dark: #6B8E77;
```
- 폰트: Sans-serif (시스템 스택). 본문 16px / 제목 24-32px / 코드 등폭.
- 반응형: 모바일 <768px (1열), 데스크톱 >1024px (2-3열).

## 구조
1. **고정 헤더**: "백엔드 개발자 류승엽" + 스티키 네비 + 다크모드 토글
2. **About**: 3-5줄 소개 + 기술 배지
3. **Main Projects (3개)**: 섹션별 상세 전개
4. **Small Projects**: 카드 나열
5. **Activities**: 연도별 이력

### Main Project 내부 순서
한줄소개 → 문제정의 → 아키텍처(플레이스홀더 800x400) → 기술선택이유 → 데이터모델 → 핵심기능 → 트러블슈팅(N개: 문제/원인/해결/결과) → 성능개선(Before/After) → 회고 → 향후개선
- 섹션 사이는 `--divider` 가는 선.
- 기술 스택: Main은 **테이블**, Small은 **배지/태그**.

## PDF 인쇄 (필수)
```css
@media print {
  .dark-toggle, .mobile-menu { display: none; }
  body { background: #fff !important; color: var(--text) !important; }
  .main-project { page-break-inside: avoid; }
  @page { size: A4; margin: 10mm; }
}
```
- Ctrl+P 시 절대 깨지면 안 됨. 다크모드는 인쇄 시 강제 라이트.

## 기술 요구사항
- 단일 HTML, embedded `<style>`, 인라인 SVG 허용.
- JS는 최소: 다크모드 토글(localStorage), 모바일 메뉴, 데이터 렌더링만.
- CSS Grid/Flexbox만 (프레임워크 금지).
- 외부 이미지 로딩 금지 (오프라인 출력 위해 플레이스홀더 사용).

## 문체 규칙
- **기호 최소화**: `—`(em dash), `·`(가운뎃점), `+` 등 기호로 개념을 이어 붙이는 방식 금지. 쉼표와 자연스러운 한국어 문장으로 대체.
- **문장 나열 위주**: "키워드 — 설명" 패턴 대신 주어·서술어가 있는 문장으로 작성.
- **예외 허용 기호**: 기술명 내 `@`, `()`, 측정값(ms, %), 다이어그램·메트릭 비교 안의 `→`.

## 작업 시 행동
- 새 프로젝트 추가 → JSON 배열에 객체 1개 추가만.
- 색상 변경 → `:root` 변수만.
- "전체 다시 줘" 요청 없으면 변경분만 출력.
